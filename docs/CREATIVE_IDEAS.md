# MyStats 창의적 기능 기획서

> "Most apps track what you do. MyStats decodes who you are."
> 이 기획은 그 철학을 극단까지 밀어붙인다.

---

## 🧬 IDEA 1: Shadow Profile (그림자 프로필)

### 컨셉
AI가 사용자가 **쓰지 않은 것**을 분석한다.

사람은 저널에 자기가 편한 것만 쓴다. 하지만 진짜 돌파구는 **회피하는 영역**에 있다.
30개의 저널에서 커리어, 관계, 돈, 건강, 창의성, 정체성 중 어떤 영역이
**통계적으로 비정상적으로 적게** 언급되는지 감지한다.

### 작동 방식
1. 저널 전체를 6개 인생 영역(Career, Relationship, Finance, Health, Creativity, Identity)으로 분류
2. 각 영역의 언급 빈도를 백분율로 계산
3. **기대 분포 대비 현저히 낮은 영역** = Shadow Zone
4. AI가 Shadow Zone에 대해 "당신이 이 영역을 피하는 이유"를 프로필 기반으로 추론

### 왜 5%인가
대부분의 저널 앱은 "쓴 것"을 분석한다. "쓰지 않은 것"의 부재를 감지하는 것은
**역추론(abductive reasoning)** 영역이다. 심리치료에서 사용하는 기법을
소프트웨어로 구현하는 앱은 거의 없다.

### UX
- Profile 페이지 하단에 "Shadow Zone" 섹션
- 6개 영역의 레이더 차트 (빈 곳이 그림자)
- 각 Shadow Zone에 AI가 쓴 1-2문장 추론
- "이 영역에 대해 써보기" → Journal로 연결

### 기술 노트
- 기존 `analyzeEntryWithAI` 프롬프트에 6영역 태깅 추가
- Insight 스키마에 `domains: string[]` 필드 추가
- 누적 통계는 프로필 로딩 시 클라이언트에서 계산

## 🔮 IDEA 2: Decision Oracle (결정 오라클)

### 컨셉
인생의 큰 결정 앞에서, AI가 **당신의 성격 프로필을 시뮬레이션**하여
각 선택지의 결과를 예측한다.

"이 회사로 이직할까?" → AI가 당신의 archetype, pattern, trait을 기반으로
**각 시나리오에서 당신이 어떻게 느끼고 행동할지** 시뮬레이션한다.

### 작동 방식
1. 사용자가 결정 상황 + 2-3개 선택지 입력
2. AI가 프로필 데이터를 기반으로 각 선택지에 대해:
   - **6개월 후 당신**: 이 선택을 한 당신의 상태 시뮬레이션
   - **충돌 지점**: 당신의 어떤 특성과 충돌하는가
   - **숨겨진 동기**: 이 선택에 끌리는 진짜 이유 (프로필 기반 추론)
   - **후회 확률**: 당신의 패턴 기반 후회 가능성
3. 최종 "Oracle Verdict" = 추천이 아니라 **거울** ("당신은 이미 답을 알고 있다")

### 왜 5%인가
의사결정 도구는 많지만, **사용자의 심리 프로필을 시뮬레이션 변수로 쓰는** 도구는 없다.
대부분은 장단점 나열이지, "이 선택이 당신의 archetype과 어떻게 상호작용하는가"를
분석하지 않는다. 이건 **개인화된 시나리오 플래닝**이다.

### UX
- Strategy 페이지에 "Decision Mode" 토글
- 결정 상황 + 선택지 입력 폼 (최대 3개)
- 각 선택지 카드에 시뮬레이션 결과
- 카드 간 비교 뷰 (어떤 archetype이 활성화되는지)

### 기술 노트
- Strategy의 변형. 기존 `generateStrategy` 프롬프트를 decision mode로 분기
- Solution 스키마에 `type: 'strategy' | 'decision'` 추가
- 프로필 + memU 컨텍스트 동일하게 활용

## ⏳ IDEA 3: Temporal Identity Drift (시간축 정체성 변이)

### 컨셉
시간에 따라 당신의 archetype, pattern, skill이 **어떻게 변했는지** 시각화한다.
3개월 전의 "나"와 지금의 "나"는 같은 사람인가?

### 작동 방식
1. Insight를 시간순으로 정렬
2. 월별/분기별로 archetype 빈도, pattern 변화, skill 증감을 추적
3. **Drift Score** 계산: 이전 기간 대비 프로필 변화량 (0-100)
4. **Phase Detection**: 안정기(plateau), 전환기(shift), 위기(disruption) 자동 감지
5. AI가 각 Phase에 대해 "이 시기에 무슨 일이 있었는지" 추론

### 왜 5%인가
대부분의 자기분석 앱은 **현재 스냅샷**만 보여준다.
시간축 위에서 **정체성의 변이를 추적**하는 것은 종단 연구(longitudinal study)의 영역이다.
이걸 개인 앱에서 자동으로 하는 건 거의 없다.

"당신은 6개월 전에 Builder였는데, 지금은 Explorer로 전환 중입니다"
— 이런 인사이트는 코치에게 돈을 내야 들을 수 있다.

### UX
- Profile 페이지에 "Timeline" 탭 추가
- 가로 타임라인 + 세로 archetype/pattern 레이어
- Drift Score 그래프 (면적 차트)
- Phase 라벨 (안정기/전환기/위기) 자동 표시
- 특정 Phase 클릭 → 해당 기간 저널 목록

### 기술 노트
- 기존 Insight 데이터 활용 (timestamp 기반 그룹핑)
- 추가 AI 호출 불필요 — 클라이언트 사이드 통계 계산
- 차트 라이브러리: recharts (이미 React 에코시스템)

## 🪞 IDEA 4: Adversarial Self-Interview (적대적 자기 인터뷰)

### 컨셉
AI가 당신의 프로필을 완전히 이해한 상태에서,
**당신이 가장 피하고 싶은 질문**을 던진다.

일반 코칭 질문("목표가 뭐예요?")이 아니다.
당신의 archetype, hidden pattern, critical question을 교차 분석해서
**당신의 방어 메커니즘을 정확히 겨냥하는 질문**을 생성한다.

### 작동 방식
1. 프로필 전체 데이터를 AI에 주입
2. AI 역할: "Socratic Adversary" — 소크라테스식 문답의 적대적 버전
3. 5개의 질문을 생성하되, 각 질문은:
   - **표면**: 단순해 보이는 질문
   - **타겟**: 이 질문이 겨냥하는 프로필 요소 (archetype/pattern/trait)
   - **의도**: 이 질문이 드러내려는 것
4. 사용자가 답변하면, 그 답변도 분석 → 후속 질문 생성 (대화형)

### 왜 5%인가
AI 챗봇은 많지만, 대부분 **사용자를 기분 좋게** 하려 한다.
사용자의 심리 프로필을 기반으로 **의도적으로 불편한 질문**을 설계하는 것은
소크라테스 대화법 + 심리 프로파일링의 교차점이다.
이건 AI가 "친절한 비서"가 아니라 **지적 도전자**가 되는 패러다임 전환이다.

### UX
- 새 페이지 또는 Profile 내 "Challenge" 모드
- 한 번에 1개 질문 표시 (전체화면, 미니멀)
- 타이핑으로 답변 → AI가 답변 분석 후 다음 질문
- 5라운드 후 "Debrief" — 인터뷰에서 드러난 인사이트 요약
- Debrief 결과를 Insight로 자동 저장

### 기술 노트
- 새로운 AI 프롬프트 체인 (multi-turn)
- Journal과 유사한 저장 흐름, type: 'interview'
- memU에 인터뷰 결과도 저장

## 🧪 IDEA 5: Paradox Portfolio (모순 포트폴리오)

### 컨셉
사용자의 프로필에서 **서로 모순되는 특성 쌍**을 찾아내고,
그 모순을 **경쟁 우위**로 재해석한다.

예: "완벽주의자인데 즉흥적" → "높은 기준을 가진 어댑터"
예: "내향적인데 리더십" → "깊이 있는 관찰자 리더"

### 작동 방식
1. 프로필의 trait, archetype, pattern을 벡터화
2. **의미적 반대 관계**에 있는 쌍을 감지 (memU 임베딩 활용)
3. 각 모순 쌍에 대해 AI가:
   - **긴장**: 이 둘이 왜 충돌하는지
   - **통합**: 이 둘이 함께 있을 때 생기는 고유한 능력
   - **활용법**: 이 모순을 어디서 무기로 쓸 수 있는지
4. "Paradox Score" = 모순의 풍부함 (높을수록 창의적 잠재력↑)

### 왜 5%인가
대부분의 프로파일링 도구는 모순을 **오류**로 처리한다 (MBTI에서 E/I 중 하나만 선택).
모순을 **특징**으로 재해석하는 것은 변증법적 사고이며,
칼 융의 "합일 (Coincidentia Oppositorum)" 개념의 소프트웨어 구현이다.

### UX
- Profile 페이지에 "Paradox" 섹션
- 각 모순 쌍을 Venn 다이어그램으로 표시 (겹치는 부분 = 통합)
- Paradox Score 배지 (1-5단계)
- "이 모순으로 전략 짜기" → Strategy로 연결

### 기술 노트
- memU embedded 엔진의 코사인 유사도를 **역방향**으로 활용 (낮은 유사도 = 높은 모순)
- 기존 프로필 데이터만으로 계산 가능
- AI 호출은 모순 쌍별 해석에만 사용

## 📡 IDEA 6: Cognitive Fingerprint (인지 지문)

### 컨셉
저널의 **내용이 아닌 형식**을 분석한다.

문장 길이, 질문 빈도, 감정어 비율, 추상적/구체적 비율, 시제 사용,
"그런데/하지만" 같은 전환어 패턴 — 이것이 당신의 **인지 지문**이다.

같은 이야기를 해도 ENTP는 ISTJ와 완전히 다르게 쓴다.
AI가 내용을 몰라도 글쓰기 패턴만으로 사고방식을 역추론할 수 있다.

### 작동 방식
1. 저널 전체에서 **언어학적 특징** 추출 (AI 불필요, 규칙 기반):
   - 평균 문장 길이 / 문단 길이
   - 질문문 비율 (? 빈도)
   - 감정어 밀도
   - 추상어 vs 구체어 비율
   - 과거/현재/미래 시제 분포
   - 접속어 패턴 (역접 vs 순접 비율)
   - 단정적 표현 vs 유보적 표현 비율
2. 이 벡터를 "Cognitive Fingerprint"로 시각화
3. AI가 fingerprint를 해석: "당신은 추상적 사고자이면서 미래 지향적이고, 단정을 피하는 경향이 있습니다"
4. 시간에 따른 fingerprint 변화도 추적 (Idea 3과 연동)

### 왜 5%인가
NLP에서 **stylometry(문체 분석)** 는 저자 식별에 쓰이지만,
이것을 **자기 이해 도구**로 전환한 소비자 앱은 없다.
"당신이 쓰는 방식이 당신의 사고방식을 드러낸다"는 통찰을
개인 앱에서 제공하는 것은 학술 연구 수준의 접근이다.

### UX
- Profile 페이지에 "Cognitive Style" 섹션
- 레이더 차트: 6-8개 인지 축
- 자연어 해석: "당신의 글쓰기는 탐색형 사고자의 전형적 패턴입니다"
- 시간별 변화 그래프 (Idea 3과 통합 가능)

### 기술 노트
- **AI 호출 최소화** — 대부분 규칙 기반 텍스트 분석 (정규식 + 통계)
- 해석 단계에서만 AI 사용
- 한국어/영어 각각 다른 분석 규칙 필요 (한국어는 어미 패턴 분석)

## 🎭 IDEA 7: Future Self Letter (미래 자아의 편지)

### 컨셉
AI가 **6개월 후의 당신**이 되어 현재의 당신에게 편지를 쓴다.

프로필의 trajectory(궤적)를 외삽하여, 지금의 패턴이 계속되면
6개월 후의 당신은 어떤 사람이 되어 있을지 시뮬레이션하고,
그 "미래의 나"가 현재의 나에게 조언한다.

### 작동 방식
1. Temporal Identity Drift(Idea 3) 데이터를 기반으로 변화 궤적 계산
2. AI에 전체 프로필 + 궤적 데이터 주입
3. AI 역할: "6개월 후의 {사용자}" — 1인칭 시점
4. 편지 내용:
   - "지금 네가 걱정하는 건..."
   - "네가 아직 모르는 건..."
   - "지금 당장 해야 할 한 가지는..."
5. 편지를 봉인 → 6개월 후 알림으로 실제 비교 가능

### 왜 5%인가
"미래의 나에게 쓰는 편지" 서비스는 있지만,
**AI가 나의 프로필을 기반으로 미래의 나를 시뮬레이션해서 편지를 쓰는** 것은
완전히 다른 차원이다. 이것은 "예측적 자기 대화"라는 새로운 카테고리다.

### UX
- Profile 또는 Home에서 "미래의 나에게서 편지가 왔습니다" 카드
- 봉투 열기 애니메이션
- 편지 형식 (수기 폰트, 날짜, 서명)
- "6개월 후 확인" 봉인 옵션 (PWA 알림)
- 이전 편지 아카이브

### 기술 노트
- Solution 스키마 활용 (type: 'future-letter')
- PWA 알림으로 6개월 후 리마인더
- Idea 3(Temporal Drift)이 있으면 더 정확, 없어도 프로필만으로 가능

## 우선순위 매트릭스

| 아이디어 | 임팩트 | 구현 난이도 | AI 비용 | 기존 인프라 활용도 | 추천 순서 |
|----------|--------|-------------|---------|-------------------|-----------|
| Shadow Profile | ★★★★★ | 중 | 중 | 높음 | 1st |
| Paradox Portfolio | ★★★★☆ | 중 | 중 | 높음 (memU) | 2nd |
| Temporal Identity Drift | ★★★★☆ | 하 | 없음 | 높음 | 3rd |
| Decision Oracle | ★★★★★ | 중 | 높음 | 높음 (Strategy 변형) | 4th |
| Cognitive Fingerprint | ★★★★☆ | 중 | 낮음 | 중간 | 5th |
| Adversarial Interview | ★★★★★ | 상 | 높음 | 중간 (새 흐름) | 6th |
| Future Self Letter | ★★★☆☆ | 하 | 중 | 높음 | 7th |

### 추천 구현 순서 근거

1. **Shadow Profile** — 기존 분석 파이프라인에 필드 하나 추가로 시작 가능. "쓰지 않은 것 분석"이라는 컨셉이 바이럴 포텐셜 최고.
2. **Paradox Portfolio** — memU 임베딩 이미 있음. 모순을 강점으로 재해석하는 프레임이 SNS 공유에 적합.
3. **Temporal Drift** — AI 호출 불필요, 기존 Insight 데이터만으로 구현. 사용자 리텐션 핵심 ("나의 변화를 보고 싶어서 계속 쓴다").
4. **Decision Oracle** — Strategy 페이지의 자연스러운 확장. 실용적 가치 최고.
5. **Cognitive Fingerprint** — 규칙 기반이라 비용 낮음. "남들은 못 보는 것을 보여준다"는 차별화.
6. **Adversarial Interview** — 가장 혁신적이지만 multi-turn 대화 구현 필요. 별도 페이지/흐름.
7. **Future Self Letter** — 감성적 가치 높지만 기능적 깊이는 낮음. 나중에 Polish 단계에서.

## 포지셔닝 전략

이 7개 아이디어가 모두 구현되면, MyStats는 더 이상 "저널 앱"이 아니다.
